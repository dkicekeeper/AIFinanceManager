## ✅ Что сделано

### 1. Отложенная загрузка при старте
**Файлы**:
- `Tenra/ViewModels/TransactionsViewModel.swift`
- `Tenra/ViewModels/AppCoordinator.swift`
- `Tenra/Views/ContentView.swift`

**Проблема**: `TransactionsViewModel.init()` синхронно загружал все данные из UserDefaults в главном потоке, блокируя запуск приложения на 2-4 секунды.

**Решение**:
- Убрал синхронный вызов `loadFromStorage()` из `init()`
- Создал асинхронный метод `loadDataAsync()` для загрузки данных в фоне
- Добавил метод `AppCoordinator.initialize()` для централизованной инициализации
- Добавил `.task {}` в `ContentView` для вызова асинхронной инициализации
- Показывается красивый skeleton loader во время загрузки

**Результат**:
- Запуск приложения: **мгновенный** (UI показывается сразу)
- Загрузка данных: **в фоне, не блокирует UI**
- Пользователь видит skeleton loader вместо белого экрана

### 2. Фоновые вычисления summary
**Файл**: `Tenra/ViewModels/TransactionsViewModel.swift`

**Проблема**: Метод `summary()` выполнял тяжелые вычисления синхронно:
- Проход по всем отфильтрованным транзакциям
- Конвертация валют для каждой транзакции
- Прогноз recurring transactions на 2 года вперед
- Общее время: ~500-1000ms для 19000 транзакций

**Решение**:
- Изменил `summary()` для использования кеша и возврата немедленного результата
- Создал приватный метод `computeSummary()` для асинхронных вычислений в фоне
- Использовал `Task.detached(priority: .userInitiated)` для фоновых вычислений
- Использовал кешированные конвертации валют (из Фазы 2) вместо синхронных

**Результат**:
- UI не блокируется во время вычислений
- Первый вызов возвращает пустой/кешированный summary мгновенно
- Вычисления происходят в фоне с приоритетом `.userInitiated`
- После завершения кеш обновляется и UI автоматически обновляется

### 3. Дебаунсинг фильтров
**Файл**: `Tenra/Views/HistoryView.swift`

**Проблема**: Каждое изменение фильтра счета или категории немедленно вызывало `updateCachedTransactions()`, что при быстром переключении фильтров приводило к множественным пересчетам.

**Решение**:
- Добавил `@State private var filterTask: Task<Void, Never>?` для отслеживания дебаунс-задач
- Изменил `.onChange(of: selectedAccountFilter)` для дебаунсинга на 150ms
- Изменил `.onChange(of: transactionsViewModel.selectedCategories)` для дебаунсинга на 150ms
- Отменяется предыдущая задача при новом изменении фильтра

**Результат**:
- Избежание множественных пересчетов при быстром переключении фильтров
- Снижение нагрузки на CPU при работе с фильтрами
- Более плавный UI без лишних обновлений

### 4. Оптимизация generateRecurringTransactions
**Файл**: `Tenra/ViewModels/TransactionsViewModel.swift`

**Проблема**: 
- Метод вызывался при каждом запуске приложения
- Отсутствовала проверка на необходимость генерации

**Решение**:
- Добавил проверку на наличие активных recurring series (пропуск, если их нет)
- Переместил вызов в `loadDataAsync()` в фоновый поток
- Добавил логирование с `PerformanceProfiler` для отслеживания производительности
- Метод уже генерирует транзакции только на 3 месяца вперед (не на 2 года)

**Результат**:
- Метод не выполняется, если нет активных recurring series
- Не блокирует запуск приложения (выполняется в фоне)
- Легко отслеживать производительность через логи

## 📊 Технические детали

### Асинхронная инициализация

```swift
// TransactionsViewModel
func loadDataAsync() async {
    await MainActor.run {
        isLoading = true
    }
    
    // Load data in background
    await Task.detached(priority: .userInitiated) {
        await MainActor.run {
            self.loadFromStorage()
        }
    }.value
    
    // Generate recurring transactions
    await Task.detached(priority: .utility) {
        await MainActor.run {
            self.generateRecurringTransactions()
        }
    }.value
    
    await MainActor.run {
        isLoading = false
    }
}
```

### Фоновые вычисления summary

```swift
func summary(timeFilterManager: TimeFilterManager) -> Summary {
    // Return cached if valid
    if !summaryCacheInvalidated, let cached = cachedSummary {
        return cached
    }
    
    // Compute in background
    Task.detached(priority: .userInitiated) {
        let result = await self.computeSummary(timeFilterManager: timeFilterManager)
        await MainActor.run {
            self.cachedSummary = result
            self.summaryCacheInvalidated = false
        }
    }
    
    // Return cached or empty summary immediately
    return cachedSummary ?? Summary(...)
}
```

### Дебаунсинг фильтров

```swift
.onChange(of: selectedAccountFilter) { oldValue, newValue in
    HapticManager.selection()
    filterTask?.cancel()
    filterTask = Task {
        try? await Task.sleep(nanoseconds: 150_000_000) // 150ms
        guard !Task.isCancelled else { return }
        await MainActor.run {
            updateCachedTransactions()
        }
    }
}
```

## ⚠️ Важные замечания

1. **Первый запуск**: При первом вызове `summary()` возвращается пустой или старый кешированный результат. UI может показать "0" на мгновение, пока не завершатся фоновые вычисления.

2. **Кеш summary**: Инвалидируется при изменении транзакций, счетов, фильтров времени. Нужно убедиться, что `invalidateCaches()` вызывается во всех нужных местах.

3. **Дебаунсинг**: Задержка 150ms может быть заметна при одиночном переключении фильтра. Но это лучше, чем множественные обновления при быстром переключении.

4. **Skeleton loading**: Показывается только при первом запуске (`isInitializing = true`). При последующих обновлениях данных скелетоны не показываются.

5. **Фоновые задачи**: `Task.detached` создает новые задачи, которые не отменяются автоматически при закрытии View. Но поскольку это вычисления summary, это приемлемо.
