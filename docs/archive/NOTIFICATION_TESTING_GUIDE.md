# 🧪 Руководство по тестированию системы уведомлений

## Быстрый старт

### 1. Подготовка к тестированию

**Сброс разрешений** (для чистого теста):
```bash
# На симуляторе
xcrun simctl privacy booted reset notifications com.yourapp.Tenra

# Или просто удалить и переустановить приложение
```

**Включить Debug View**:
Добавьте в Settings или главное меню:
```swift
#if DEBUG
NavigationLink("🔔 Notification Debug") {
    NotificationDebugView()
}
#endif
```

---

## 2. Тест №1: Запрос разрешений

### Шаги:
1. Запустить приложение (первый раз после установки)
2. Перейти в "Подписки" → "+"
3. Заполнить данные подписки
4. Включить любое напоминание (например, "За 3 дня")
5. Нажать "Сохранить"

### Ожидаемый результат:
✅ Появляется sheet `NotificationPermissionView` с текстом "Включить напоминания?"
✅ При нажатии "Разрешить" → iOS показывает системный диалог
✅ После разрешения → sheet закрывается, подписка сохраняется

### Как проверить:
- Открыть **Notification Debug View**
- **Permission Status** должен быть `Authorized` (зеленый)

---

## 3. Тест №2: Правильность расчета даты

### Шаги:
1. Создать подписку:
   - Название: "Netflix"
   - Сумма: 9.99
   - Частота: Ежемесячно
   - Дата начала: **15 февраля 2026**
   - Напоминания: 1 день, 3 дня

2. Открыть **Notification Debug View**
3. Нажать "Refresh" в секции "Pending Notifications"

### Ожидаемый результат:
✅ Должно быть **2 запланированных уведомления**:
- Уведомление 1: 📅 **14 марта 2026** (3 дня до 15 марта)
- Уведомление 2: 📅 **14 марта 2026** (1 день до 15 марта)

### Логика:
- Сегодня: 14 февраля 2026
- Start Date: 15 февраля 2026
- Next Charge: 15 марта 2026 (следующий месяц, 15 число)
- Напоминание "за 3 дня": 15 марта - 3 = **12 марта**
- Напоминание "за 1 день": 15 марта - 1 = **14 марта**

## 7. Тест №6: Пауза подписки

### Шаги:
1. Создать подписку с напоминаниями
2. Проверить, что уведомления запланированы
3. Открыть детальный вид подписки
4. Меню (⋯) → "Приостановить"
5. Вернуться в **Notification Debug View**

### Ожидаемый результат:
✅ Уведомления для этой подписки **удалены**
✅ В консоли: `🗑️ [Scheduler] Cancelled notifications for: subscription_XXX`

## 8. Тест №7: Удаление подписки

### Шаги:

3. Открыть детальный вид подписки
4. Меню (⋯) → "Удалить" → выбрать любой вариант

✅ Уведомления для этой подписки **удалены**

---

## 9. Тест №8: Статус разрешений

### Проверка всех статусов:

**Not Determined** (не запрошено):
- Свежая установка → должен быть "Not Determined"

**Denied** (отклонено):
- Settings → AI Finance Manager → Notifications → OFF
- В Debug View → "Denied" (красный)

**Authorized** (разрешено):
- Settings → AI Finance Manager → Notifications → ON
- В Debug View → "Authorized" (зеленый)

## ❌ Типичные проблемы

### Проблема: Уведомления не появляются
**Решение**:
1. Проверить Permission Status в Debug View
2. Проверить Settings → Notifications → AI Finance Manager
3. **На симуляторе**: Уведомления могут не работать корректно → тестировать на реальном устройстве

### Проблема: Неправильная дата уведомления
**Решение**:
1. Проверить логи calculateNextChargeDate
2. Убедиться, что startDate не в прошлом
3. Проверить frequency подписки

### Проблема: Permission Request не появляется
**Решение**:
1. Проверить, создается ли **новая** подписка (не редактирование)
2. Проверить, что выбраны **reminderOffsets** (хотя бы один)
3. Проверить, что разрешение еще не запрашивалось

## ✅ Критерии успеха

Система работает корректно, если:

1. ✅ При первой подписке запрашиваются разрешения
2. ✅ Уведомления планируются для правильных дат
3. ✅ Расчет следующей даты учитывает startDate
4. ✅ При паузе/удалении уведомления отменяются
5. ✅ При перезапуске приложения уведомления перепланируются
6. ✅ Debug View показывает актуальную информацию
7. ✅ Тестовое уведомление приходит через 5 секунд
8. ✅ В логах нет ошибок

## 📱 Тестирование на реальном устройстве

**ВАЖНО**: Симулятор имеет ограничения с уведомлениями!

### Подключение устройства:
1. Подключить iPhone/iPad через USB
2. Xcode → Product → Destination → Ваше устройство
3. Запустить приложение
4. Проверить все тесты выше

### Проверка уведомлений:
- Свернуть приложение (кнопка Home)
- Подождать времени срабатывания
- Уведомление должно появиться на Lock Screen
